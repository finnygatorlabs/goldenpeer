How It Works

Example: 100 seniors ask "What's the weather in Seattle?"

1.
First user: API calls OpenWeatherMap → Response cached for 2 hours

2.
Next 99 users (within 2 hours): Return cached result (no API call)

3.
Result: 1 API call instead of 100 ✅

With 10,000 seniors:

•
Without caching: 50,000 API calls/day

•
With caching: ~1,500 API calls/day (70% reduction)




Automatic Query Detection

The system automatically knows which API to use:

Weather Detection

Triggers if query contains: "weather", "forecast", "rain", "temperature", "cold", "hot"

Sports Detection

Triggers if query contains: "score", "game", "nfl", "nba", "mlb", "nhl", "football", "basketball", "baseball", "hockey", "team", "player"

News Detection

Triggers if query contains: "news", "happened", "today", "current", "latest"

Time Detection

Triggers if query contains: "time", "timezone"

Bible Detection

Triggers if query contains: "bible", "verse", "scripture"

General Knowledge (Default)

Everything else goes to Wikipedia




Graceful Degradation

If an API goes down, users get helpful fallback messages:

If ESPN API is down:

Plain Text


User: "How did the Seahawks play?"
Response: "I'm having trouble reaching sports data right now. 
          Try asking about weather, news, or general topics instead."



If NewsData.io is down:

Plain Text


User: "What's in the news?"
Response: "News service is temporarily unavailable. 
          I can help with weather, sports, or general knowledge instead."



If all APIs are down:

Plain Text


User: "Tell me something"
Response: "I'm experiencing technical difficulties. 
          Please try again in a few minutes."
