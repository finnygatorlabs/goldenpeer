SeniorShield Scam Detection Engine Specification

Overview

The scam detection engine is the core differentiator of SeniorShield. It must be powerful, accurate, and fast. Since users can only paste/type text (not upload screenshots), we'll build a multi-layered detection system that analyzes:

1. Scam Detection Architecture

Multi-Layer Detection System

Plain Text


INPUT (User pastes text/email/message)
    ↓
LAYER 1: Text Preprocessing
    - Extract sender info (email, phone, name)
    - Extract URLs and links
    - Extract phone numbers
    - Extract financial amounts
    - Normalize text (lowercase, remove extra spaces)
    ↓
LAYER 2: Keyword Detection
    - Urgency keywords (URGENT, ACT NOW, VERIFY, CONFIRM)
    - Financial keywords (PAYMENT, TRANSFER, ACCOUNT, CARD)
    - Authority keywords (IRS, BANK, AMAZON, APPLE, MICROSOFT)
    - Threat keywords (SUSPENDED, LOCKED, COMPROMISED, FRAUD)
    ↓
LAYER 3: Pattern Matching
    - Common scam templates (phishing, spoofing, urgency)
    - Impersonation patterns (fake bank, fake government)
    - Request patterns (password, PIN, account number)
    ↓
LAYER 4: Link Analysis
    - Suspicious URL patterns
    - Domain spoofing detection
    - Shortened URL expansion
    - Known malicious domains
    ↓
LAYER 5: Sender Analysis
    - Email domain verification
    - Phone number verification
    - Known scammer databases
    ↓
LAYER 6: Contextual Analysis
    - Relationship to user (expected vs. unexpected)
    - Timing (unusual time for this contact?)
    - Consistency (does this match how they usually communicate?)
    ↓
LAYER 7: ML Scoring
    - Combine all signals
    - Calculate risk score (0-100)
    - Generate confidence level
    ↓
OUTPUT: Risk Score + Detailed Analysis + Recommendations






2. Risk Scoring Algorithm (0-100)

Scoring Matrix

Plain Text


BASE SCORE: 0 (Safe)

LAYER 1: KEYWORD DETECTION (0-30 points)
- Urgency keywords: +5 points each (max 10)
  * "URGENT", "ACT NOW", "IMMEDIATE", "TODAY", "IMMEDIATELY"
- Financial keywords: +3 points each (max 9)
  * "PAYMENT", "TRANSFER", "ACCOUNT", "CARD", "BILLING"
- Authority keywords: +4 points each (max 12)
  * "IRS", "BANK", "AMAZON", "APPLE", "MICROSOFT", "PAYPAL"
- Threat keywords: +5 points each (max 15)
  * "SUSPENDED", "LOCKED", "COMPROMISED", "FRAUD", "UNAUTHORIZED"

LAYER 2: PATTERN MATCHING (0-25 points)
- Phishing pattern detected: +15 points
  * "Verify your account", "Confirm your identity", "Update your information"
- Spoofing pattern detected: +15 points
  * Sender claims to be from known company but domain is different
- Urgency pattern detected: +10 points
  * Multiple urgency keywords + financial request
- Request for sensitive info: +20 points
  * Password, PIN, SSN, account number, credit card
- Impersonation pattern: +15 points
  * Fake bank, fake government agency, fake tech support

LAYER 3: LINK ANALYSIS (0-20 points)
- Suspicious URL pattern: +8 points
  * Shortened URL (bit.ly, tinyurl)
  * Suspicious domain (misspelled, suspicious TLD)
  * URL doesn't match sender
- Malicious domain detected: +20 points
  * Known phishing domain
  * Known malware domain
- Domain spoofing detected: +15 points
  * Domain looks like legitimate company but isn't

LAYER 4: SENDER ANALYSIS (0-15 points)
- Unknown sender: +5 points
- Sender claims to be from known company: +10 points
- Email domain doesn't match company: +15 points
- Phone number flagged as scammer: +20 points
- Sender in known scammer database: +20 points

LAYER 5: CONTEXTUAL ANALYSIS (0-10 points)
- Unexpected contact from known company: +5 points
- Unusual time for this contact: +3 points
- Inconsistent with user's history: +5 points
- User has never interacted with sender before: +3 points

MAXIMUM SCORE: 100 (Definitely a scam)

RISK LEVELS:
- 0-20: Safe (Green) ✅
- 21-40: Low Risk (Yellow) ⚠️
- 41-60: Medium Risk (Orange) ⚠️⚠️
- 61-80: High Risk (Red) 🚨
- 81-100: Critical Risk (Dark Red) 🚨🚨

4. Pattern Matching System

Common Scam Patterns

Plain Text


PATTERN 1: PHISHING
Characteristics:
- Claim to be from legitimate company
- Request to verify account/identity
- Urgent language
- Link to fake website
- Request for credentials

Detection:
- "Verify your account" OR "Confirm your identity" = +15 points
- Sender claims to be from known company = +10 points
- Suspicious link included = +8 points
- Urgency keywords present = +5 points

Example: "Your Amazon account needs verification. Click here to confirm your identity."

---

PATTERN 2: SPOOFING
Characteristics:
- Sender appears to be from legitimate company
- Email/phone number doesn't match company
- Request for sensitive information
- Urgent tone

Detection:
- Email domain doesn't match company = +15 points
- Phone number doesn't match company = +15 points
- Request for sensitive info = +20 points
- Urgency keywords = +5 points

Example: From "support@amazonsecure.net" (not amazon.com)

---

PATTERN 3: URGENCY SCAM
Characteristics:
- Multiple urgency keywords
- Financial request
- Threat of consequences
- Pressure to act immediately

Detection:
- 3+ urgency keywords = +10 points
- Financial keyword present = +3 points
- Threat keyword present = +5 points
- Request for immediate action = +5 points

Example: "URGENT: Your account will be closed TODAY if you don't verify now!"

---

PATTERN 4: IMPERSONATION
Characteristics:
- Claim to be from government agency, bank, or tech company
- Request for personal information
- Threat of legal action or account closure
- Unusual contact method

Detection:
- Claims to be from government/bank/tech = +15 points
- Request for SSN, account number, etc. = +20 points
- Threat of legal action = +10 points
- Unusual contact method (email/text for phone company) = +5 points

Example: "This is the IRS. You owe back taxes. Call immediately or face legal action."

---

PATTERN 5: TECH SUPPORT SCAM
Characteristics:
- Claim computer has virus/malware
- Urgent tone
- Request to call number or download software
- Threat of data loss

Detection:
- Claims computer is infected = +15 points
- Request to call number = +10 points
- Request to download software = +15 points
- Threat of data loss = +10 points
- Domain doesn't match company = +15 points

Example: "WARNING: Your computer has 47 viruses! Call Microsoft Support immediately!"

---

PATTERN 6: ROMANCE SCAM
Characteristics:
- Builds emotional connection
- Eventually requests money
- Story about emergency/hardship
- Request for wire transfer or gift cards

Detection:
- Emotional language = +5 points
- Request for money = +20 points
- Story about emergency = +10 points
- Request for wire/gift card = +15 points
- Unusual relationship context = +5 points

Example: "I love you so much. I'm stuck in [country]. Can you send me $5,000?"

---

PATTERN 7: LOTTERY/PRIZE SCAM
Characteristics:
- Claim user won prize/lottery
- User never entered
- Request for personal information
- Request for upfront payment

Detection:
- Claims user won = +15 points
- User never entered = +10 points
- Request for personal info = +15 points
- Request for upfront payment = +20 points

Example: "Congratulations! You won $1 million! Send $500 for processing fees."

---

PATTERN 8: GRANDPARENT SCAM
Characteristics:
- Claim to be grandchild
- Emergency situation
- Request for money
- Urgency to not tell parents

Detection:
- Claims to be family member = +10 points
- Emergency situation = +10 points
- Request for money = +20 points
- "Don't tell parents" language = +15 points
- Unusual contact method = +5 points

Example: "Grandma, it's me. I'm in jail and need $5,000 bail. Don't tell Mom!"






5. Suspicious Link Detection

URL Analysis Engine

Plain Text


SUSPICIOUS URL PATTERNS:
- Shortened URLs (bit.ly, tinyurl, etc.): +8 points
- URL doesn't match sender: +10 points
- Misspelled domain (amaz0n.com instead of amazon.com): +15 points
- Suspicious TLD (.tk, .ml, .ga, .cf): +10 points
- IP address instead of domain: +15 points
- Multiple subdomains (verify.secure.amazon-help.com): +10 points
- @ symbol in URL (user@domain.com): +15 points
- Encoded characters in URL (%2F, %3D): +8 points

MALICIOUS DOMAIN DATABASE:
- Known phishing domains (maintained by PhishTank, OpenPhish)
- Known malware domains (maintained by URLhaus)
- Known scam domains (maintained by community reports)

DOMAIN VERIFICATION:
- Check domain registration date (new domains = higher risk)
- Check domain age (domains < 30 days old = +10 points)
- Check domain reputation (Alexa, VirusTotal)
- Check SSL certificate (self-signed = +5 points)
- Check domain registrar (suspicious registrars = +5 points)

EXAMPLE ANALYSIS:
URL: "http://bit.ly/verify-amazon"
- Shortened URL: +8 points
- HTTP (not HTTPS ): +5 points
- Unknown destination: +10 points
TOTAL: 23 points (Low-Medium Risk)

URL: "https://verify.secure.amazon-help.com/login"
- Domain doesn't match Amazon: +10 points
- Misspelled domain: +15 points
- Multiple subdomains: +10 points
- HTTPS (legitimate ): +0 points
TOTAL: 35 points (Medium Risk)






6. Misspelling Detection

Typosquatting & Domain Spoofing

Plain Text


COMMON MISSPELLINGS:
- amaz0n.com (zero instead of O)
- amazn.com (missing O)
- amzon.com (missing A)
- amazom.com (extra O)
- amaon.com (missing Z)
- amazno.com (reversed)

DETECTION ALGORITHM:
1. Extract domain from email/URL
2. Compare to known legitimate domains
3. Calculate Levenshtein distance (how many edits to match)
4. If distance < 3, flag as potential spoofing

SCORING:
- 1 character difference: +12 points
- 2 character difference: +10 points
- 3 character difference: +8 points
- Homograph attack (similar looking characters): +15 points

HOMOGRAPH ATTACKS:
- rn vs m (amazon.com vs amaz0n.com)
- l vs 1 vs | (paypal.com vs paypa1.com)
- O vs 0 (microsoft.com vs microsof0.com)
- S vs 5 (bank.com vs ban5.com)

EXAMPLE:
Domain: "amaz0n.com" (with zero)
Legitimate: "amazon.com" (with O)
Levenshtein distance: 1
Score: +12 points
